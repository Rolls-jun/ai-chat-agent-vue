## 第一阶段：补齐基础体验（优先级高）✅ 已完成

### 1.1 长对话记忆压缩

**目标**：对话超过 20 轮后自动摘要，防止 token 溢出

**实施步骤**：

1. **后端：检测对话长度**
   - 在 `server/index.mjs` 的 `/api/chat` 路由中
   - 统计 `messages` 数组长度，超过 20 条触发压缩

2. **后端：实现摘要逻辑**
   - 取前 10 条 user/assistant 消息
   - 调用模型生成 200 字摘要
   - 摘要作为 system prompt 追加段落：`[对话历史摘要: ...]`
   - 后续只保留最近 10 条原文

3. **后端：缓存摘要**
   - 用 `chat-<id>-summary` 键存储摘要到 localStorage
   - 下次对话时直接读取，避免重复生成

4. **测试验证**
   - 发送 25 条消息，检查是否触发压缩
   - 验证摘要内容是否准确
   - 确认 token 使用量是否下降

---

### 1.2 设置面板

**目标**：用户可切换模型、调整温度、清除全部历史

**实施步骤**：

1. **前端：创建设置组件**
   - 新建 `src/components/SettingsModal.vue`
   - 顶部添加齿轮图标按钮，点击打开模态框

2. **前端：模型切换**
   - 下拉选择：DeepSeek-V4-Pro / GPT-4o / Claude-3.5
   - 选择后保存到 `localStorage.model`
   - 发送请求时带上 `model` 参数

3. **前端：温度调节**
   - 滑块范围 0-1，步长 0.1
   - 保存到 `localStorage.temperature`
   - 后端接收 `temperature` 参数传给 `streamText`

4. **前端：清除历史**
   - 按钮点击后确认弹窗
   - 清除所有 `chat-*` 和 `chat-list` 键
   - 刷新页面

5. **后端：接收设置参数**
   - `/api/chat` 路由解析 `model` 和 `temperature`
   - 动态创建对应模型实例

---

### 1.3 对话导出/导入

**目标**：支持 JSON 导出当前对话，导入恢复

**实施步骤**：

1. **前端：导出功能**
   - 侧边栏每个对话项添加"导出"按钮
   - 读取 `chat-<id>` 的消息数组
   - 生成 JSON 文件下载：`chat-<timestamp>.json`

2. **前端：导入功能**
   - 设置面板中添加"导入对话"按钮
   - `<input type="file" accept=".json">` 选择文件
   - 解析 JSON，生成新的 `chat-<uuid>` 键
   - 更新 `chat-list` 数组

3. **前端：数据校验**
   - 检查 JSON 格式是否正确
   - 验证 `messages` 数组结构
   - 错误时提示用户

---

### 1.4 消息操作

**目标**：复制单条消息、重新生成、编辑用户消息

**实施步骤**：

1. **前端：消息悬浮操作栏**
   - 每条消息 hover 时显示操作按钮
   - 用户消息：复制、编辑
   - AI 消息：复制、重新生成

2. **前端：复制消息**
   - 点击复制按钮 → `navigator.clipboard.writeText(text)`
   - 按钮变绿勾 2 秒后恢复

3. **前端：编辑用户消息**
   - 点击编辑 → 消息变成可编辑的 textarea
   - 保存后删除该消息之后的所有消息
   - 重新发送请求

4. **前端：重新生成**
   - 删除当前 AI 消息
   - 用相同的用户消息重新请求
   - 流式输出新回答

5. **后端：支持重新生成**
   - `/api/regenerate` 路由
   - 接收 `chatId` 和 `messageIndex`
   - 截断消息到指定索引，重新生成

---

### 1.5 移动端适配

**目标**：侧边栏可折叠、输入框自适应

**实施步骤**：

1. **前端：侧边栏折叠**
   - 移动端默认隐藏侧边栏
   - 顶部添加汉堡菜单按钮
   - 点击展开侧边栏（覆盖层）

2. **前端：响应式布局**
   - 使用 Tailwind 的 `md:` 断点
   - 移动端：`flex-col`，侧边栏绝对定位
   - 桌面端：`flex-row`，侧边栏固定宽度

3. **前端：输入框自适应**
   - 移动端输入框高度自适应
   - 发送按钮固定在右下角
   - 键盘弹起时页面不跳动

4. **测试**
   - 手机浏览器访问
   - 检查触摸事件是否正常
   - 验证横竖屏切换

## 技术栈

- **前端**：Vue 3 + Vite + Tailwind CSS v4
- **后端**：Express + Vercel AI SDK v6
- **模型**：DeepSeek-V4-Pro（通过 OpenAI 兼容 API）
- **存储**：localStorage（前端）
- **工具**：marked（Markdown 渲染）、zod（参数校验）

## 快速开始

```bash
# 安装依赖
npm install

# 启动后端（端口 3001）
npm run server

# 启动前端（端口 5173）
npm run dev
```

访问 http://localhost:5173

## 环境变量

在 `.env` 中配置：

```env
OPENAI_API_KEY=your_api_key
OPENAI_BASE_URL=https://api.deepseek.com/v1
OPENAI_MODEL=deepseek-chat
TAVILY_API_KEY=your_tavily_key  # 可选，用于联网搜索
```
